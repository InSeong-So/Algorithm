## HashTable

### 정리
```sh
키와 값이 한 쌍을 이루는 데이터를 저장한다.
선형 탐색의 경우 배열의 데이터에 비례하여 계산 시간이 늘어나는 문제를 해결하기 위한 자료구조이다.
해시 함수를 이용해 키에 해당하는 해시값을 계산, 저장할 배열의 사이즈 값만큼 나누어 나머지를 구한다.
나머지에 해당하는 인덱스에 데이터를 저장하고, 이러한 절차를 다른 데이터에도 똑같이 반복한다.
만약 나머지 연산을 통한 나머지 값이 다른 데이터와 같으면 저장 위치가 겹쳐 '충돌'이라고 한다.
이 경우, 리스트 구조로 해당 배열의 인덱스에 있는 기존 데이터와 추가할 데이터를 연결한다.
해시 테이블은 해시 함수를 이용해 배열 내의 특정 데이터에 빠르게 접근할 수 있다.
만약 해시값이 충돌한다면 추가된 리스트 구조로 유연하게 대응할 수 있다.
배열의 크기가 너무 작으면 충돌이 많아지고 선형 탐색의 빈도가 높아진다.
크기가 너무 크면 데이터가 없는 상자(공간)가 많아져 메모리를 낭비하게 된다.
데이터의 유연한 저장, 빠른 접근이 가능한 해시 테이블은 프로그래밍 언어의 연관배열에 사용된다.
```

<hr>

```sh
연쇄법(체이닝Chaining) : 충돌을 리스트 형태로 기존 데이터의 뒤에 연결하는 방법
개방 주소법(Open Addressing) : 충돌을 다음 후보가 될 주소(배열상의 위치)를 구해 저장하는 방법
나머지로는 '다음 주소'를 해시 함수를 여러개 사용하여 구현하는 방법과 '선형 검색'이 있다.
```
